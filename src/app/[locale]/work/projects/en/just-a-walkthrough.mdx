**Framework‑agnostic, zero‑dependency product tour / onboarding walkthrough with optional React helpers.**


## Features

- Spotlight highlight & darkened backdrop (3 panel overlay + focus ring)
- Accessible keyboard navigation (Esc / Enter / ← →) + focus trap
- Auto scroll + responsive reposition on scroll/resize/mutations
- Step hooks (<InlineCode>beforeStep</InlineCode> / <InlineCode>afterStep</InlineCode>) & lifecycle callbacks
- Chain multiple tours with persistence-aware skipping
- LocalStorage progress persistence & resume support
- Optional once‑per‑session logic via orchestrator helper
- Custom tooltip renderer + theming (<InlineCode>default</InlineCode>, <InlineCode>tailwind</InlineCode>, <InlineCode>unstyled</InlineCode>)
- Works with any DOM (vanilla, React, shadcn, portals)
- Tiny, tree‑shakeable (no external deps)

## API Surface (Core)

<InlineCode>startWalkthrough(steps, options)</InlineCode> – convenience wrapper returning a <InlineCode>Walkthrough</InlineCode> instance.

<InlineCode>Walkthrough</InlineCode> key methods:

- <InlineCode>start(index?)</InlineCode> – begin tour (auto called by helper)
- <InlineCode>next()/prev()</InlineCode>
- <InlineCode>finish()</InlineCode> – mark completed & cleanup
- <InlineCode>skip(reason?)</InlineCode>
- <InlineCode>destroy()</InlineCode> – cleanup without marking completed

## Theming

Use <InlineCode>theme: 'tailwind'</InlineCode> to rely on your Tailwind stack (supply utility classes) or <InlineCode>unstyled</InlineCode> to supply all styling manually.

## Advanced Usage Notes

- Use <InlineCode>waitMs: 0</InlineCode> (or global <InlineCode>stepWaitMs: 0</InlineCode>) for elements that are guaranteed to be present to avoid unnecessary polling.
- Prefer chaining tours when you have progressive disclosure flows; persistent tours auto-skip if already completed, keeping chains idempotent.
- When dynamically removing highlighted elements mid-step (e.g. route transitions), the MutationObserver repositions but if the element disappears the next navigation call will resolve again. Consider guarding with required steps if element is critical.

## Accessibility Notes

- Focus ring container traps tab order (unless <InlineCode>disableFocusTrap: true</InlineCode>)
- Live region announces step titles (<InlineCode>aria-live="polite"</InlineCode>)
- Esc always available (when <InlineCode>keyboard: true</InlineCode>)

## Zero-Wait Configuration


Specify <InlineCode>stepWaitMs: 0</InlineCode> and/or per-step <InlineCode>waitMs: 0</InlineCode> to disable polling for elements (single lookup). <InlineCode>stepPollIntervalMs</InlineCode> is clamped to a minimum of 1ms internally when waiting.
